###### Gestione dei blocchi liberi 
Anche la gestione dei blocchi liberi viene fatta attraverso diverse tecniche abbiamo già visto in passato:
- *bitmap*
- *liste concatenate*
a seguito di crash del sistema i file system possono diventare inconsistenti se lavoro su un file system già corrotto posso fare ancora più danno, apposite *utility* possono effettuare dei *controlli di consistenza* sui blocchi. Uno dei tanti check che si possono fare per capire se un file system è incoerente è quello di controllare la struttura dati che gestisce i blocchi liberi, se questa è coerente (ovvero i blocchi liberi sono veramente liberi) allora il file system è coerente.


Controllo di consistenza img d (due file puntano alla stessa locazione di memoria): la soluzione a quel problema è quella di creare un copia del blocco di memoria facendo in modo che i due file puntino a due blocchi di memoria diversi (ma che contengono la stessa cosa) - uno dei due file è sicuramente danneggiato starà all'operatore controllare quale dei due file è rotto

un operazione idempotente: è un operazione che posso ripetere due volte senza creare problemi
